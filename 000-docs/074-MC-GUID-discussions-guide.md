# Discussion Categories

This repository encourages collaboration through GitHub Discussions. Please visit our [Discussions page](https://github.com/jeremylongshore/claude-code-plugins-nixtla/discussions) to participate.

## Recommended Discussion Categories

When setting up discussions, we recommend these categories to encourage collaboration:

### 1. Plugin Ideas
**Description:** Propose new Claude Code plugin concepts for Nixtla integration
**Format:** Ideas
- Share your vision for new plugins
- Discuss potential use cases
- Collaborate on specifications

### 2. Show and Tell
**Description:** Share your Nixtla workflows and how plugins could improve them
**Format:** Show and Tell
- Demonstrate current Nixtla implementations
- Share success stories
- Showcase generated code

### 3. Technical Q&A
**Description:** Ask technical questions about plugin development and Nixtla integration
**Format:** Q&A
- Get help with implementation
- Discuss technical challenges
- Share solutions

### 4. Collaboration Opportunities
**Description:** Find collaborators for plugin development projects
**Format:** General
- Team up on plugin development
- Share expertise
- Coordinate contributions

### 5. Feature Requests
**Description:** Request features for existing plugin concepts
**Format:** Ideas
- Suggest enhancements
- Discuss implementation approaches
- Vote on priorities

### 6. Use Cases & Examples
**Description:** Share real-world use cases and example code
**Format:** General
- Document practical applications
- Share code snippets
- Discuss best practices

## How to Participate

1. **Start a Discussion:** Click "New discussion" and choose the appropriate category
2. **Be Specific:** Provide clear examples and context
3. **Share Code:** Use markdown code blocks for clarity
4. **Collaborate:** Engage with others' ideas and offer constructive feedback

## Collaboration Guidelines

- **Open Source Spirit:** All plugin concepts are MIT licensed
- **Respectful Communication:** Professional and constructive dialogue
- **Credit Contributors:** Acknowledge others' ideas and contributions
- **Focus on Value:** Prioritize plugins that solve real problems
