# Nixtla Model Selector

Automatically selects the best forecasting model for a given time series dataset.

## Purpose

To streamline the model selection process by automatically identifying the most suitable forecasting model (StatsForecast or TimeGPT) based on the characteristics of the input data.

## Overview

Analyzes time series data to determine whether StatsForecast or TimeGPT would yield more accurate forecasts. This skill considers factors such as data length, frequency, seasonality, and presence of outliers. It then executes the chosen model and returns the forecasts. This eliminates the need for manual model selection and experimentation. Outputs the forecasts generated by the selected model.

## Prerequisites

**Tools**: Read, Write, Bash, Glob, Grep

**Environment**: `NIXTLA_TIMEGPT_API_KEY` (required if TimeGPT is selected)

**Packages**:
```bash
pip install statsforecast nixtla pandas
```

## Instructions

### Step 1: Load and analyze data

Read input CSV and analyze its characteristics (length, frequency, seasonality, outliers).

### Step 2: Select model

Determine whether StatsForecast or TimeGPT is better suited based on data characteristics.

### Step 3: Execute forecast

Run the selected model (StatsForecast or TimeGPT) on the input data.

### Step 4: Generate output

Save the forecasts to a CSV file.

## Output

- **forecast.csv**: Predictions generated by the selected model.
- **model_selection.txt**: Text file indicating which model was selected and why.

## Error Handling

1. **Error**: `Input file not found`
   **Solution**: Verify the file path and ensure the file exists.

2. **Error**: `Invalid data format`
   **Solution**: Ensure the input CSV contains the required columns (unique_id, ds, y).

3. **Error**: `NIXTLA_TIMEGPT_API_KEY not set and TimeGPT selected`
   **Solution**: Set the environment variable `NIXTLA_TIMEGPT_API_KEY` before running the script.

4. **Error**: `No model could be selected`
   **Solution**: Check if the input data is valid and meets the minimum requirements for both StatsForecast and TimeGPT.

## Examples

### Example 1: Short, seasonal data

**Input**:
```
unique_id,ds,y
product_1,2023-01-01,10
product_1,2023-01-02,12
product_1,2023-01-03,15
```

**Output**:
- **forecast.csv**: (StatsForecast predictions)
- **model_selection.txt**: "StatsForecast selected due to short data length and presence of seasonality."

### Example 2: Long, non-seasonal data

**Input**:
```
unique_id,ds,y
location_1,2010-01-01,100
location_1,2010-01-02,102
location_1,2010-01-03,105
... (Many more rows)
```

**Output**:
- **forecast.csv**: (TimeGPT predictions)
- **model_selection.txt**: "TimeGPT selected due to long data length and lack of clear seasonality."
