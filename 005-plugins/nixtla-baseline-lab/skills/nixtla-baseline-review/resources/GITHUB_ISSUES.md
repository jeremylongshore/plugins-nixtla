# GitHub Issue Draft Guide

## Overview

The GitHub issue draft feature helps users share baseline results with Nixtla maintainers or the statsforecast community with complete reproducibility information.

## When to Suggest GitHub Issue Drafts

**User scenarios**:
- Wants to ask Nixtla maintainers a question about statsforecast
- Suspects a bug or unexpected behavior
- Wants to share benchmark results with the Nixtla community
- Asks "How do I share these results with Nixtla?"
- Reports performance issues or surprising metrics

## How to Create Issue Drafts

### Step 1: Choose Issue Type

Use the `generate_github_issue_draft` tool with appropriate `issue_type`:

**`"question"` (default)**:
- For community support questions
- "Why does AutoETS perform worse on these series?"
- "What season_length should I use for this pattern?"

**`"bug"`**:
- For suspected bugs or unexpected behavior
- "AutoTheta crashes on series with missing values"
- "MASE calculation seems incorrect for hourly data"

**`"benchmark"`**:
- For sharing performance results
- "AutoTheta significantly outperforms on M4-Daily subset"
- "Comparing statsforecast to TimeGPT on custom dataset"

### Step 2: Auto-Detection

The tool automatically detects and includes:
- **Metrics CSV**: Complete results table
- **Benchmark report**: If generated via `generate_benchmark_report`
- **Compatibility info**: Library versions from `compat_info.json`
- **Run manifest**: Configuration from `run_manifest.json`

### Step 3: Output Location

Issue draft is written to:
```
nixtla_baseline_m4_{timestamp}/github_issue_draft.md
```

## What the Issue Draft Includes

### Standard Template Components

**Header**:
- Issue type tag (question/bug/benchmark)
- Title placeholder for user to fill in
- Context about the Nixtla Baseline Lab plugin

**User Description Section**:
- Placeholder for user's specific question/bug/results
- Reminder to fill this in before posting

**Complete Reproducibility Bundle**:
- Run configuration (dataset, horizon, models, freq, season_length)
- Benchmark results (if report exists)
- Library versions (statsforecast, datasetsforecast, pandas, numpy)
- Timestamps and demo preset (if used)

**Footer**:
- Attribution to Nixtla Baseline Lab Claude Code plugin
- Links to plugin documentation (if public)

## Typical Workflow

### Recommended Sequence
```python
# 1. Run baselines (repro bundle auto-generated by default)
run_baselines(demo_preset="m4_daily_small")

# 2. Generate benchmark report (recommended before issue draft)
generate_benchmark_report()

# 3. Create issue draft with appropriate type
generate_github_issue_draft(issue_type="question")

# 4. User reviews github_issue_draft.md and posts to GitHub
```

### Why Generate Report First?
- Provides formatted, copy-paste ready results
- Includes highlights and insights
- Makes the issue more readable for maintainers
- Shows you've done your homework

## Example User Prompts

**"Help me create a GitHub issue to ask Nixtla about these sMAPE values"**
- Use `issue_type="question"`
- Include benchmark report if not already generated
- Remind user to describe their specific question

**"Generate an issue draft reporting this AutoETS bug"**
- Use `issue_type="bug"`
- Emphasize importance of repro bundle
- Ask user to describe bug symptoms clearly

**"Create a benchmark issue to share these results with Nixtla"**
- Use `issue_type="benchmark"`
- Must have benchmark report already generated
- Highlight key findings in the draft

## Important Reminders for Users

### This is a Community Plugin
- **Not official Nixtla tooling**: Mention you're using "Nixtla Baseline Lab Claude Code plugin"
- **Be respectful**: Maintainers are volunteers or have limited bandwidth
- **Self-service first**: Check existing issues and documentation before posting

### Reproducibility is Key
- **Include all info**: Versions, parameters, dataset details
- **Attach artifacts**: If possible, attach metrics CSV
- **Be specific**: "AutoETS fails on series D10" not "AutoETS doesn't work"

### Where to Post
- **statsforecast issues**: For model behavior, bugs, performance questions
- **datasetsforecast issues**: For dataset loading or preprocessing issues
- **nixtla SDK issues**: For TimeGPT-related questions

### Etiquette
- Search for duplicates first
- Use descriptive titles
- Be patient with responses
- Update issue if you find a solution

## Advanced: Custom Issue Types

If user needs something beyond question/bug/benchmark:

**Fallback**:
- Generate with default `issue_type="question"`
- User manually edits the template
- Keep the reproducibility bundle intact

**Feature request**:
- Suggest posting to Nixtla's feature request tracker (if they have one)
- Include context about why the feature would help

## Validation Before Posting

**Checklist to share with user**:
- [ ] Filled in description/question/bug details
- [ ] Verified metrics and configuration are correct
- [ ] Checked for duplicate issues on GitHub
- [ ] Used descriptive title (not "help" or "question")
- [ ] Removed any sensitive data (if using custom dataset)
- [ ] Added appropriate labels (bug, question, enhancement)
- [ ] Mentioned this is from "Nixtla Baseline Lab Claude Code plugin"

## When to Discourage GitHub Issues

**Don't suggest GitHub for**:
- Basic usage questions (point to documentation)
- Plugin-specific issues (not Nixtla's responsibility)
- Very preliminary exploration (run more experiments first)
- Already answered in FAQ or docs

**Instead**:
- Link to statsforecast documentation
- Suggest checking plugin README
- Recommend running more series or different configurations
- Point to Nixtla's community Slack/Discord (if they have one)

## Error Handling

**If repro bundle is missing**:
```
To create a GitHub issue draft with complete reproducibility info,
please re-run baselines with generate_repro_bundle=True:

/nixtla-baseline-m4 demo_preset=m4_daily_small generate_repro_bundle=true
```

**If metrics CSV is missing**:
```
Cannot generate issue draft without baseline results.
Please run /nixtla-baseline-m4 first to generate metrics.
```

**If user hasn't reviewed draft**:
```
I've created the draft at: nixtla_baseline_m4_{timestamp}/github_issue_draft.md

Please review and fill in:
- Your specific question/bug description
- Any additional context
- A descriptive title

Then copy-paste to: https://github.com/Nixtla/statsforecast/issues/new
```
